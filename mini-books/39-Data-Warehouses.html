Data Warehouses: O Grande Salão do Tesouro Otimizado para Análise na Nuvem! 🏛️✨📊

Olá de novo, Desbravador(a) analítico(a)!

O Data Warehouse é o GRANDE SALÃO DE EXIBIÇÃO 🏛️✨ dos seus dados: um repositório centralizado, integrado e otimizado para análise, onde os tesouros (dados) já vêm polidos pela Engenharia de Analytics e são organizados para facilitar consultas rápidas e complexas pelos analistas e sistemas de BI.

Analogie: DATA WAREHOUSE = O GRANDE SALÃO DE EXIBIÇÃO 🏛️✨, onde os tesouros polidos de várias origens são arrumados para análise rápida e fácil.

Mental Trigger: DW = Salão para Análise Rápida 🏛️📊.

O Que é um Data Warehouse? O Salão de Análise Otimizado! 📊🏛️

  • Centraliza e integra dados de várias fontes para análise e decisão.
  • OLAP: Otimizado para consultas complexas e históricas. (vs. OLTP, que é transacional)
  • Características: Orientado por assunto, integrado, histórico (com tempo), não volátil.

Modelagem no Salão: O Layout Perfeito (Dimensional)! 🖼️💎

  • Tabelas Fato: Medidas quantitativas (valores, contagens) + FKs para dimensões.
  • Tabelas Dimensão: Contexto descritivo (quem, o quê, quando, onde).
  • Schema Estrela: Fato no centro, dimensões em volta (layout simples e eficiente).
  • Schema Floco de Neve: Dimensões normalizadas em subdimensões (mais complexo, economiza espaço).
Estrela: Um display central com alas para cada contexto.
Floco de Neve: Alas que se ramificam em salas menores.

ETL/ELT no Salão: Como os Dados Chegam!

  • Dados vêm de fontes (S3, OLTP...)
  • Polidos/transformados pela Engenharia de Analytics (Glue, dbt, etc.)
  • Carregados nas tabelas Fato e Dimensão do DW
  • Prontos para dashboards e KPIs!

AWS Redshift: O Grande Salão Gerenciado na Nuvem ☁️🏛️

  • Gerenciado: AWS cuida da infra e manutenção.
  • Armazenamento Colunar: Guarda dados por coluna, não linha — super rápido para análises.
  • MPP: Processamento massivamente paralelo para consultas rápidas em grandes volumes.
  • Integração: Perfeito com S3, Glue, QuickSight e demais serviços AWS.
  • Custo-efetivo: Paga pelo uso (cluster ou serverless).

Data Warehouse vs. Data Lake vs. Banco de Dados 🏛️📦🏦

  • Banco OLTP: Operações do dia a dia, transações rápidas. (🏦)
  • Data Lake: Dados brutos, multi-formato, barato, schema on read. (☁️📦)
  • Data Warehouse: Dados estruturados, integrados, otimizados para análise. Schema on write. (🏛️✨📊)
  • Fluxo comum: Data Lake ➡️ ETL/ELT ➡️ Data Warehouse ➡️ Dashboards/BI

Caso de Uso: Dashboard de KPIs Históricos 📊🎯

  1. Dados de vendas chegam ao S3 (Data Lake).
  2. Engenharia de Analytics (Glue/dbt) limpa, transforma e carrega no Redshift (DW) em schema dimensional.
  3. QuickSight conecta no Redshift para dashboards de KPIs históricos com performance!

Recapitulando o Grande Salão! 🧠🏛️✨📊

  • DW: Repositório central para análise e decisão.
  • Otimizado para OLAP: Consultas pesadas, modelagem dimensional.
  • Carregamento: Via ETL/ELT.
  • Redshift: DW gerenciado, colunar, integrado à AWS.
  • Papel: Destino para dados prontos para análise em escala.
