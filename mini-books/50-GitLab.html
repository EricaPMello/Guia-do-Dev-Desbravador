GitLab: O Quartel General Completo da Frota de Desbravadores (Além do Diário de Bordo!) 🛠️🚢🗺️✅

Olá de novo, Desbravador(a) que busca o quartel general perfeito!

GitLab é uma plataforma baseada em Git que vai além do controle de versão: integra ferramentas para todo o ciclo de vida do desenvolvimento de software e de projetos de dados, incluindo planejamento, automação, deploy e monitoramento — tudo em um só lugar!

Analogie: Se o GitHub é o Diário de Bordo e a Biblioteca de Mapas 📜🗺️🤝, o GITLAB é o QUARTEL GENERAL COMPLETO 🛠️🚢🗺️✅ — com oficinas, painéis de comando, salas de planejamento e oficinas automatizadas!

Mental Trigger: GITLAB = Quartel General DevOps 🛠️🚢.

Por Que Conhecer o Quartel General GitLab?

  • Alternativa popular ao GitHub, especialmente para DevOps integrado e self-hosting.
  • CI/CD integrado forte: Oficinas automatizadas fazem build, test e deploy a cada atualização.
  • Cobertura do Ciclo de Vida: Planejamento (Issues, Boards), código, testes, deploy, monitoramento, tudo integrado.
  • Self-hosted: Instale em servidores próprios se quiser mais controle.
  • Merge Requests (MRs): Equivalente ao Pull Request do GitHub.

Vocabulário GitLab 🗣️🛠️

  • Project: Repositório Git (pasta do projeto).
  • Commit: Um registro de mudanças.
  • Branch: Linha paralela de desenvolvimento.
  • Merge Request (MR): Proposta de mesclar mudanças de uma branch para outra (igual ao PR do GitHub).
  • Groups: Organização de projetos.
  • CI/CD Pipelines: Sequência de jobs automatizados definidos em .gitlab-ci.yml.
  • Issues & Boards: Gerenciamento de tarefas e fluxo de trabalho.
  • Wiki: Documentação colaborativa.
O .gitlab-ci.yml é o manual das oficinas automatizadas: define build, test, deploy!

GitLab na Prática do Desbravador de Dados 💻📊🛠️

  • Versione scripts, notebooks, queries, pipelines e modelos.
  • Use Merge Requests para revisão de código.
  • Automatize testes, ETL, deploy de ML e dashboards com CI/CD.
  • Planeje tarefas com Issues e Boards.

GitLab vs. GitHub

  • GitHub: Foco em comunidade open source, marketplace, CI/CD via Actions.
  • GitLab: DevOps integrado, pipelines nativos, forte em self-hosting.
  • Ambos usam Git, commits, branches, MRs (PRs).

Caso de Uso: Pipelines ETL Automatizados no GitLab CI/CD ✨📦🚚✅

  1. Código ETL em Python/PySpark no repositório.
  2. .gitlab-ci.yml define estágios (build, test, deploy).
  3. Teste automatizado do código e dos dados de exemplo.
  4. Deploy automatizado: aciona Glue, carrega dados no Redshift, atualiza dashboards.
  5. Pipelines podem rodar por atualização de código ou por agendamento.
